Bienvenue à la journée de contribution !

Cette clé USB contient les trois logiciels dont vous aurez besoin pour rapidement monter un environnement de développement local pour contribuer à WordPress :

  • Virtualbox - un logiciel d’exécution de machines virtuelles.
  • Vagrant - un logiciel de création et de configuration de machines virtuelles.
  • VVV - un environnement Vagrant dédié au développement WordPress.

Plus d’informations au sujet de VVV sont disponibles sur VaryingVagrantVagrants.org.

Instructions d’installation

Important : les utilisateurs de Microsoft Windows auront besoin d’exécuter les commandes en utilisant leurs droits d’administrateur. Pour cela vous pouvez utiliser le clic droit sur le fichier cmd.exe et sélectionner « Exécuter en tant qu’Administrateur ».

  1. Créez un répertoire sur votre ordinateur pour y copier les fichiers nécessaires.
  2. Copiez-y le contenu des répertoires windows ou mac (en fonction de votre équipement) depuis la clé USB.
  3. Copiez-y les fichiers vvv.zip, vagrant-hostsupdater.gem et vvv-contribute.box depuis la clé USB.
  4. Copiez-y ce fichier d’instructions afin de pouvoir vous y référer si vous devez donner cette clé USB à quelqu’un d’autre.
  5. Donnez cette clé USB aux participants de la journée de contribution qui en auraient besoin.
  1. Installez Virtualbox en utilisant l’installateur du répertoire windows ou mac.
    • En cas d’échec sur Mac, lancez les préférences système, et ouvrez les réglages de « Securité et de confidentialité »
    • Vous devriez constater la présence d’un message à proximité du bas de la fenêtre indiquant qu’un logiciel édité par Oracle America Inc a été bloqué. Vous avez besoin de l’autoriser.
    • Une fois que c’est fait, relancez l’installateur de VirtualBox et tout devrait bien se dérouler.
    • Lisez ceci pour plus d’informations sur le sujet
  2. Installez Vagrant en utilisant l’installateur du répertoire windows ou mac.
  3. Ouvrez l’utilitaire de lignes de commande/le terminal et naviguez jusqu‘au répertoire où vous avez copié les fichiers à l’aide de la commande cd.
  4. Ajoutez l’environment VVV à Vagrant : 
    vagrant box add vvv/contribute vvv-contribute.box
  5. Installez le gem Vagrant Hosts Updater : 
    vagrant plugin install vagrant-hostsupdater.gem
  6. Décompressez vvv.zip.
  7. Naviguez à présent dans le répertoire vvv qui a été décompressé. Assurez vous qu’il contient bien le fichier Vagrantfile.
  8. Démarrez Vagrant pour la première fois : 
    vagrant up --no-provision --provider virtualbox
  9. Visitez http://vvv.test dans votre navigateur Internet.

Vous pouvez désormais utiliser vagrant halt pour interrompre l’environnement, et vagrant up pour le relancer.

Connexion

Cette copie pré-provisionnée de VVV intègre plusieurs instances de WordPress.

2 installations standards de WordPress sont disponibles depuis http://one.wordpress.test et http://two.wordpress.test

Pour contribuer au noyau de WordPress, vous utiliserez http://trunk.wordpress.test.

Pour contribuer à WordPress Meta, vous disposerez de toutes les versions de développement des sites Meta listés ici.

Pour chacune de vos instances WordPress, les éléments d’authentification à l’administration sont les suivants :

  • Identifiant : admin
  • Mot de passe : password

Lisez la page des élément d’authentification par défaut de la documentation de VVV pour plus d’informations sur l’authentification et les emplacements en local ou dans la machine virtuelle des fichiers.

Provisionnement

Re-provisionner est nécessaire à l’ajout ou à la suppression de sites, mais attention, cette opération est très consommatrice et peut ruiner la connexion wifi de tous les participants de cette journée.

Si vous souhaitez poursuivre l’expérimentation de VVV pour le développement local de vos sites ou de ceux de vos clients, merci d’attendre d’être rentré chez vous !
